**Step B — Create app/[locale]/dashboard/listings/new/page.tsx:**

This is a "use client" page. Structure:

1. On mount (useEffect):
   - Read `localStorage.getItem("auth_token")` → if null, redirect to `/${locale}/login`
   - Fetch `/api/auth/me` with Bearer header → if role !== "SELLER", redirect to `/${locale}/dashboard`
   - Fetch `/api/categories` → populate category select options
   - Fetch `/api/products?sellerId=me&limit=100` with Bearer header → count activeListings

2. State:
   - `titleEn`, `titleAr`, `descEn`, `descAr`, `price`, `imageUrl`, `categoryId`, `platform`, `deliveryType` ("INSTANT" | "MANUAL")
   - `activeCount: number`, `submitting: boolean`, `error: string`, `imagePreview: string`

3. SEC-05 banner: when `activeCount >= 10`, render:
   ```tsx
   <div className="bg-amber-500/10 border border-amber-500/20 rounded-xl px-4 py-3 mb-6 text-amber-400 text-sm">
     You've reached your 10-listing limit. Complete your first sale to unlock unlimited listings.
   </div>
   ```

4. Form layout (max-w-2xl mx-auto card with bg-card/60 backdrop-blur-sm border border-white/8 rounded-2xl p-8):

   a. EN/AR tabs using shadcn Tabs component:
   ```tsx
   <Tabs defaultValue="en">
     <TabsList><TabsTrigger value="en">English</TabsTrigger><TabsTrigger value="ar">العربية</TabsTrigger></TabsList>
     <TabsContent value="en">
       <div className="space-y-4">
         <div>
           <label htmlFor="titleEn" className="text-xs font-medium text-muted-foreground uppercase tracking-wider mb-1 block">Title (EN)</label>
           <Input id="titleEn" value={titleEn} onChange={e => setTitleEn(e.target.value)} className="rounded-xl" required />
         </div>
         <div>
           <label htmlFor="descEn">Description (EN)</label>
           <Textarea id="descEn" value={descEn} onChange={e => setDescEn(e.target.value)} rows={4} className="rounded-xl" />
         </div>
       </div>
     </TabsContent>
     <TabsContent value="ar">
       <!-- same fields with dir="rtl" on inputs, id="titleAr"/"descAr" -->
     </TabsContent>
   </Tabs>
   ```

   b. Language-agnostic fields:
   - Price: number input with `$` prefix label, `min="0.01"`, `step="0.01"`
   - Category: `<Select>` from shadcn populated with categories from API
   - Platform: `<Select>` with PLATFORMS constant options
   - Delivery Type: `<RadioGroup>` with two options:
     - INSTANT: "Instant Key" — "Buyer receives the license key immediately"
     - MANUAL: "Manual Delivery" — "You post the credentials after receiving the order"
   - Image URL: text input, `onBlur` sets imagePreview; show `<img src={imagePreview} alt="preview" className="mt-2 h-16 w-16 rounded-lg object-cover" />` if preview URL is non-empty

   c. Form validation: all required fields validated before submit. Show `text-xs text-red-400 mt-1` error below each invalid field.

   d. Submit button:
   ```tsx
   <Button
     onClick={handleSubmit}
     disabled={submitting || activeCount >= 10}
     className="w-full rounded-xl bg-accent text-accent-foreground font-bold py-3"

     {submitting ? <><Loader2 className="w-4 h-4 mr-2 animate-spin" />Creating...</> : "Create Listing"}
   </Button>
   ```

   - Validate: titleEn, titleAr, descEn, descAr all non-empty; price > 0; deliveryType set
   - POST `/api/products` with Bearer header and body `{ title: { en: titleEn, ar: titleAr }, description: { en: descEn, ar: descAr }, price: Number(price), imageUrl: imageUrl || undefined, categoryId: categoryId || undefined, platform: platform || undefined, deliveryType }`
   - On 201: `router.push(`/${locale}/products/${data.id}`)` (per D-11)
   - On 403 SEC-05: show error banner
   - On other error: show error message

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client form → POST /api/products | Seller role verified server-side; sellerId injected server-side |
| Client → PUT /api/products/[id] | Seller ownership verified via updateMany({ where: { id, sellerId } }) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-15 | Spoofing | sellerId in listing creation | mitigate | API injects sellerId from JWT; client form never sends sellerId |
| T-02-16 | Elevation of Privilege | Seller editing another seller's listing | mitigate | PUT uses productService.update(id, sellerId) — updateMany returns count=0 on mismatch → 403 |
| T-02-17 | Elevation of Privilege | SEC-05 bypass via direct API call | mitigate | Cap enforced server-side in POST /api/products (02-03); client banner is cosmetic only |
