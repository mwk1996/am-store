**key.service.ts — add encryption to bulkCreate():**

Import encryptKey:
  import { encryptKey } from "@/lib/crypto";

Update bulkCreate():
  async bulkCreate(productId: string, keyValues: string[]) {
    // Encrypt each key before storing (SEC-02)
    const encrypted = keyValues.map((kv) => ({
      productId,
      keyValue: encryptKey(kv),  // ← AES-256-GCM ciphertext
    }));
    return prisma.productKey.createMany({
      data: encrypted,
      skipDuplicates: true,
    });
  }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → POST /api/products | Seller role required; sellerId injected server-side |
| API response → client | keyValue must never appear in any product response |
| POST /api/products/[id]/keys | Seller must own the product; keys encrypted before storage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-06 | Spoofing | sellerId in POST /api/products body | mitigate | Reject client-supplied sellerId; inject from JWT user.userId server-side |
| T-02-07 | Elevation of Privilege | SEC-05 listing cap | mitigate | Server-side count check before create; client badge is cosmetic only |
| T-02-08 | Information Disclosure | keyValue in product list/detail | mitigate | productInclude uses _count only, never keys relation; grep-verified |
| T-02-09 | Tampering | Seller A updating Seller B's product | mitigate | productService.update uses updateMany({ where: { id, sellerId } }); count=0 returns 403 |
| T-02-10 | Information Disclosure | keyValue in key import API response | mitigate | POST /api/products/[id]/keys returns only { created: count }; no key values |
</threat_model>
