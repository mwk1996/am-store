<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → checkout form | User-supplied productId goes into order creation — must be validated server-side |
| Browser → key reveal component | Auth token sent as Authorization header on every key fetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-08-01 | Spoofing | Checkout — unauthenticated order creation | mitigate | Auth redirect at page mount + API returns 401 without token |
| T-03-08-02 | Information Disclosure | Key reveal — token stored in localStorage | accept | Consistent with existing dashboard auth pattern; httpOnly cookie migration is Phase 4+ |
| T-03-08-03 | Tampering | Checkout — client sends gateway not in allowed list | mitigate | Server-side Zod enum validation on /api/payment/initiate rejects unknown gateways |
| T-03-08-04 | Information Disclosure | Key visible before payment confirmed | mitigate | Key reveal button only shown when order.status in [PAID, COMPLETED]; PENDING state shows spinner |
</threat_model>
