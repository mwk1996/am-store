Run the following commands in sequence. Each must succeed before the next runs.

**Step 1 — Validate schema is complete (precondition check):**
```bash
npx prisma validate
```
If this fails, Plan 01 (schema changes) did not complete correctly. Stop and fix schema first.

**Step 2 — Run migration (interactive TTY available):**
```bash
npx prisma migrate dev --name purchase-delivery
```
This will:
- Detect schema changes (new Order fields, AuditLog model, AuditEvent enum, OrderStatus FAILED, User back-relations)
- Generate SQL migration file in prisma/migrations/
- Apply migration to the database
- Regenerate Prisma client

**Step 3 — If DATABASE_URL is not available (CI / non-TTY environment), use db push instead:**
```bash
npx prisma db push
npx prisma generate
```

**Step 4 — Verify Prisma client was regenerated:**
```bash
npx prisma generate
```
Running generate a second time is safe (idempotent) and confirms the client is up to date.

**Environment note:** Ensure `DATABASE_URL` is set in `.env` before running. If running locally with a dev PostgreSQL instance, the migration will succeed immediately. If DATABASE_URL is missing, the executor must stop and ask the developer to set it.
