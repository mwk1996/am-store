# Phase 3 — UI Design Contract: Purchase & Delivery

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (manual install — no components.json, but components/ui/ is populated) |
| Preset | not applicable |
| Component library | Radix UI (via shadcn/ui) |
| Icon library | lucide-react |
| Font | Inter (Google Fonts, variable weight 100–900) |

Existing shadcn components available: `Button`, `Badge`, `Card`, `Dialog`, `Input`, `Label`, `RadioGroup`, `Select`, `Table`, `Tabs`, `Textarea`, `Toast`, `Toaster`

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, step-number gap |
| sm | 8px | Compact element spacing, inline form hints |
| md | 16px | Default element spacing, card internal padding sections |
| lg | 24px | Card padding (`p-6`), section breaks within a card |
| xl | 32px | Gap between major card columns |
| 2xl | 48px | Vertical page padding (`py-12`) |
| 3xl | 64px | Reserved for page-level top spacing on wide viewports |

Exceptions:
- Touch targets for gateway selector tiles: minimum 44px height
- Key reveal box: `py-5 px-6` (20px / 24px) for comfortable key reading
- Progress indicator steps: `py-1 px-3` (4px / 12px) pill size — intentionally compact

---

## Typography

Two weights only: **400** (regular) for body, captions, hints, key value; **600** (semibold) for all emphasized text — labels, headings, CTAs, display/page title.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 |
| Label / UI text | 14px (`text-sm`) | 600 (semibold) | 1.25 |
| Heading (card/section) | 16px (`text-base`) | 600 (semibold) | 1.3 |
| Display (page title) | 30px (`text-3xl`) | 600 (semibold) | 1.2 |

Supporting sizes:
- Caption / hint text: 12px (`text-xs`) at weight 400, line-height 1.25
- Key value (monospace): 14px (`font-mono text-sm`) at weight 400
- Badge label: 12px (`text-xs`) at weight 600

Font family: `'Inter'` with `font-feature-settings: "cv02", "cv03", "cv04", "cv11"` — already applied globally via `globals.css`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(222 47% 6%)` — `bg-background` | Page background |
| Secondary (30%) | `hsl(222 40% 9%)` — `bg-card` / `hsl(222 40% 13%)` — `bg-secondary` | Cards, sidebar, input fields, table rows |
| Accent (10%) | `hsl(25 95% 53%)` — `bg-accent` (orange) | Primary CTA buttons only (Buy Now, Pay, Confirm Receipt, Deliver) |
| Primary (highlight) | `hsl(217 91% 60%)` — `text-primary` / `border-primary` | Trust badges, progress step highlight, key box border, link text |
| Destructive | `hsl(0 84% 60%)` — `bg-destructive` | Cancel order action only |

Accent (`hsl(25 95% 53%)`) is reserved **only** for:
1. "Buy Now" / "Pay [amount] IQD" submit button on checkout
2. "Confirm Receipt" button on buyer order detail (DELIVERED state)
3. "Deliver" submit button inside seller delivery modal
4. Wallet balance "Pay with Wallet" button (when active)

All other interactive elements (nav links, secondary actions, reveal toggle) use `primary` blue or `secondary` surface styling.

Order status badge colors (established in Phase 2 — do not change):
- `PENDING` — `text-amber-400 bg-amber-500/10`
- `PAID` — `text-blue-400 bg-blue-500/10`
- `DELIVERED` — `text-purple-400 bg-purple-500/10`
- `COMPLETED` — `text-emerald-400 bg-emerald-500/10`
- `DISPUTED` — `text-red-400 bg-red-500/10`
- `REFUNDED` — `text-gray-400 bg-gray-500/10`

## Screen-by-Screen Contract

### 1. Checkout Page (`/[locale]/checkout`)

**Layout:** `max-w-3xl mx-auto` two-column grid on md+: form left (3/5), order summary right (2/5). Single column on mobile.

**Card shell:** `rounded-2xl border border-white/8 bg-card/60 backdrop-blur-sm p-6` (matches established pattern — note: existing checkout uses `rounded-xl`; Phase 3 standardizes to `rounded-2xl` per the phase context card style).

**Gateway selector grid:**
- Rendered as a `grid grid-cols-2 sm:grid-cols-3` of selectable tiles, not a dropdown
- Each tile: `rounded-xl border border-border/60 bg-secondary/50 p-4 cursor-pointer transition-all`
- Selected state: `border-primary/60 bg-primary/10 ring-1 ring-primary/40`
- Hover state: `border-border hover:border-primary/30`
- Content: gateway logo/icon + name label (`text-xs font-semibold text-center`)
- Use `RadioGroup` from shadcn/ui as the underlying control
- Gateways: QI Card, ZainCash, FIB, Asia Pay, Fast Pay (5 tiles)
- Disabled gateway tile (for future toggle): `opacity-40 cursor-not-allowed pointer-events-none`

**Wallet balance option:**
- Shown as a 6th tile only when buyer has sufficient wallet balance
- Label: "Wallet Balance — [amount] IQD"
- If balance is insufficient: tile shown but disabled with tooltip "Insufficient balance"

**Order summary card (right column):**
- Product name at `text-sm font-semibold`
- Delivery type indicator: `<Badge>` with `Zap` icon for INSTANT, `Clock` icon for MANUAL
- Price line + total line separated by `border-t border-border/60 pt-4`
- Trust row: `ShieldCheck` icon + "Secured by [gateway name]" at `text-xs text-muted-foreground`

**Progress indicator:** 3-step pill row centered above the card grid — "Details" (step 1 active), "Payment" (step 2 inactive), "Receive" (step 3 inactive). Use existing pattern from `checkout/page.tsx` exactly.

**Primary CTA:** "Pay [price] IQD" (e.g., "Pay 15,000 IQD") — full width, accent background, `rounded-xl py-3.5 text-sm font-semibold`. Loading state: spinner + "Processing...". Disabled when no gateway selected.

---

### 2. Payment Processing / Redirect Feedback

**Page:** Shown briefly before redirect to gateway, or as a full-screen interstitial if redirect takes > 1s.

**Layout:** Centered, `min-h-screen flex flex-col items-center justify-center`

**Content:**
- Animated spinner (CSS `animate-spin` on a `Loader2` lucide icon, `h-10 w-10 text-primary`)
- Heading: "Redirecting to payment..." at `text-lg font-semibold`
- Sub-copy: "You will be redirected to [Gateway Name] to complete your payment. Do not close this page." at `text-sm text-muted-foreground text-center max-w-sm`

**Error state (gateway initiation failed):**
- Replace spinner with `AlertCircle` icon in `text-destructive`
- Heading: "Payment could not be started"
- Body: "There was a problem connecting to [Gateway Name]. Your order has not been charged." + "Try again" button (outline variant) + "Choose a different gateway" link

---

### 3. Order Success / Key Reveal Page (`/[locale]/orders/[id]`)

**Layout:** `max-w-2xl mx-auto py-12 px-4` centered single column.

**Success header:**
- `CheckCircle2` icon (`h-12 w-12 text-emerald-400`) centered with `animate-fade-in`
- Heading: "Payment Successful" at `text-2xl font-semibold`
- Sub: "Your order has been confirmed." at `text-sm text-muted-foreground`

**Key reveal box (INSTANT delivery):**
- Card: `rounded-2xl border border-primary/20 bg-primary/5 backdrop-blur-sm p-6`
- Section label: "Your License Key" at `text-xs font-semibold uppercase tracking-wider text-muted-foreground mb-3`
- Key value: `font-mono text-sm font-normal` inside `rounded-lg border border-border/60 bg-secondary/60 px-4 py-3 break-all select-all`
- Copy button: icon-only `Button` variant `ghost` with `Copy` lucide icon, positioned `absolute top-2 right-2` relative to the key box. On click: icon changes to `Check` for 2 seconds, toast "Copied to clipboard"
- Email notice: `text-xs text-muted-foreground` — "We've also sent this key to [email]"

**Manual delivery status (MANUAL delivery — awaiting):**
- Status card with `Clock` icon (`text-amber-400`), heading "Awaiting Delivery", body "The seller has 24 hours to deliver your account credentials." at `text-sm text-muted-foreground`
- No key box shown — replaced by delivery status timeline (see §5)

**Order metadata row:** Product name, order ID (truncated `...` after 12 chars), date, price — displayed as a simple `grid grid-cols-2 gap-2 text-sm` list inside a secondary card below the key box.

**Next actions row:** Two ghost buttons — "View Order History" and "Continue Shopping" — centered, `gap-4`.

---

### 4. Buyer Order History Page (`/[locale]/orders`)

**Layout:** `max-w-4xl mx-auto py-10 px-4`

**Page heading:** "My Orders" at `text-2xl font-semibold` + order count badge `text-xs bg-secondary rounded-full px-2 py-0.5`

**Empty state:**
- `ShoppingBag` icon (`h-12 w-12 text-muted-foreground/40`) centered
- Heading: "No orders yet" at `text-base font-semibold`
- Body: "Your purchases will appear here after checkout." at `text-sm text-muted-foreground`
- CTA button: "Browse Products" → `/[locale]` (primary variant, not accent — this is a secondary action)

**Order list:** Table on md+, card stack on mobile.

Table columns: Product | Type | Status | Date | Price | Action
- Product: thumbnail (32×32 `rounded-md object-cover`) + name + category badge
- Type: `Zap` (INSTANT) or `Clock` (MANUAL) icon with label `text-xs text-muted-foreground`
- Status: `<Badge>` using `STATUS_COLORS` map from established pattern
- Date: formatted `dd MMM yyyy` at `text-xs text-muted-foreground`
- Price: `text-sm font-semibold`
- Action: context-sensitive (see below)

**Row action — context by status:**
- PAID / COMPLETED + INSTANT: "Reveal Key" → ghost `Button` with `Key` icon. On click: inline expand below the row showing key box (same pattern as §3). No navigation.
- DELIVERED + MANUAL: "Confirm Receipt" → accent `Button` small. Shows confirmation dialog before API call.
- PENDING: "View Details" → ghost link
- COMPLETED + MANUAL: "View Details" → ghost link (credentials already confirmed)
- DISPUTED / REFUNDED: Status badge only, no action button

**Loading skeleton:** 5 rows of shimmer `<tr>` with `h-4 rounded shimmer` cells — same shimmer class from globals.css.

---

### 5. Manual Delivery Status Timeline (on order detail / order history row)

Used for MANUAL delivery orders. Shown as a vertical 3-step stepper inside the order card or expanded row.

**Steps:**
1. "Order Placed" — always completed (`CheckCircle2 text-emerald-400`)
2. "Awaiting Delivery" → "Delivered" — completed when seller posts credentials (`CheckCircle2 text-emerald-400` / `Clock text-amber-400` pending)
3. "Receipt Confirmed" — completed when buyer confirms or auto-confirms (`CheckCircle2 text-emerald-400` / ghost when pending)

**Visual:** Vertical connector line `w-px h-8 bg-border/60 ml-3` between steps. Each step: `flex items-start gap-3`, icon `h-4 w-4 shrink-0 mt-0.5`, label `text-sm font-semibold`, sublabel `text-xs text-muted-foreground`.

**Confirm Receipt CTA** (step 3 pending): Accent button "Confirm Receipt" + helper text "Auto-confirms in [N] hours if you don't respond." Opens a `Dialog` with title "Confirm Receipt?", body "By confirming, you verify you've received the account credentials and the seller will be paid.", two buttons: "Cancel" (outline) and "Confirm Receipt" (accent).

**Credentials display** (DELIVERED state only): Same key box pattern as §3 — monospace font, copy button — shown after step 2 completes. Label changed to "Account Credentials".

---

### 6. Seller Orders Tab (Dashboard → Orders)

**Location:** Extends `app/[locale]/dashboard/page.tsx` — adds `orders` tab value to existing `ActiveTab` type.

**Tab label:** "Orders" with unread/pending badge count (amber pill `text-xs`) if pending orders exist.

**Order list within tab:** Table using existing `<Table>` component.
Columns: Buyer (email truncated) | Product | Type | Status | Date | Action

**Pending MANUAL delivery rows** — highlighted with `border-l-2 border-amber-400/60 bg-amber-500/5` on the `<tr>`.

**"Deliver" button:** Accent `Button` size `sm` with `Send` lucide icon — only shown on PENDING rows for MANUAL delivery products.

**Deliver modal:**
- `Dialog` with title "Deliver Credentials"
- Product name subtitle at `text-sm text-muted-foreground mb-4`
- `Textarea` placeholder: "Enter account credentials exactly as the buyer should receive them.\n\nExample:\nEmail: user@example.com\nPassword: abc123XYZ"
- Character minimum: 10 (validated client-side with inline error `text-xs text-destructive`)
- Preview panel below textarea: `rounded-lg border border-border/40 bg-secondary/40 p-3 text-xs font-mono text-foreground` — label "Buyer will see:" above it
- Submit button: accent "Deliver Now" full width. Loading: "Delivering..." + spinner. Disabled when textarea empty or < 10 chars.
- Cancel: outline "Cancel" — closes dialog without submitting

**Auto-confirm indicator:** For DELIVERED rows past 12h: amber `Clock` icon + "Auto-confirms in [N]h" text at `text-xs text-muted-foreground`. Past 20h: changes to `text-amber-400` to draw attention.

## Interaction States

| State | Pattern |
|-------|---------|
| Loading (page data) | Shimmer skeleton using `.shimmer` utility class |
| Loading (button action) | `Loader2` lucide icon `animate-spin h-4 w-4` inline before label text, button disabled |
| Success (copy to clipboard) | `Copy` → `Check` icon swap for 2s, `useToast` with "Copied!" message |
| Success (delivery confirmed) | Toast "Receipt confirmed. Order completed." + row status badge updates optimistically |
| Success (credentials delivered) | Toast "Credentials delivered successfully." + modal closes + row status updates |
| Error (API) | `useToast` destructive variant: "[Action] failed. [Reason]. Please try again." |
| Empty (no orders) | Illustrated empty state per §4 — no table rendered |
| Disabled gateway tile | `opacity-40 cursor-not-allowed` with `title` tooltip explaining why |

## RTL / i18n Contract

- All layout uses Tailwind's `rtl:` variant for directional spacing: `ms-*` / `me-*` instead of `ml-*` / `mr-*` where direction-sensitive
- Gateway selector grid: same grid layout regardless of direction (grid is not directional)
- Key box copy button: `end-2 top-2` (logical properties) not `right-2`
- Progress indicator arrows: use `rtl:rotate-180` on `ArrowRight` icons
- All display text sourced from `useTranslations()` — no hardcoded strings
- Product names via `getLocalizedText(title, locale)` with `en` fallback
- Arabic font: Inter supports Arabic — no additional font needed
