Harden all `app/api/admin/*` routes by replacing inline `verifyToken + role check` patterns with the `requireAdminSession()` helper from `lib/auth-middleware.ts`. This closes SEC-07: middleware-only protection is insufficient in App Router — every admin route needs an explicit per-route session check.

Purpose: Eliminates the security gap where a route without middleware coverage would have no auth enforcement.
Output: 8 admin route files updated to use requireAdminSession.
</objective>
